activate(transition) [-> Promise]

调用时机在它被创建，而且马上要进行过渡阶段的时候

  • 参数 Arguments transition {Transition} 调用 transition.next()来解决此钩子函数，注意：这里调用 transition.abort()的话，将不会返回到之前的路径，因为此时过渡已经通过了验证---但是demo6中点击中止过渡还是回去了，详情Transition Hooks
  • 返回值 Return Value 可以选择返回一个 Promise
    • resolve(data) -> transition.next(data)
    • reject(reason) -> transition.abort(reason)
  • 详细信息 Detail

    大多数情况下，此钩子函数用来控制视图切换的时机，因为 此函数如果没执行完，切换是不会开始的。

    此钩子函数的顺序是从上到下的，子视图组件的activate只会在父视图组件的activate执行完毕之后被调用
