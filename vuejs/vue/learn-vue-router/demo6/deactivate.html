deactivate(transition) [-> Promise]

调用时机在视图它将要被停用，并移除的时候

  • 参数 Arguments transition {Transition} 和 active一样

    调用 transition.next()来解决此钩子函数，注意：这里调用 transition.abort()的话，将不会返回到之前的路径，因为此时过渡已经通过了验证

  • 返回值 Return Value 可以选择返回一个 Promise
    • resolve(data) -> transition.next(data)
    • reject(reason) -> transition.abort(reason)
  • 详细信息 Detail

    此钩子函数被调用顺序是从下往上的，父视图组件的deactivate钩子函数，只会在其子视图组件的 deactivate调用完毕后执行

    新视图组件的 activate 钩子函数，只会在 所有 当前停用的组件的 deactivate钩子执行完毕后调用
